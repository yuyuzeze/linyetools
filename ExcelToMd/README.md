# Excel → Markdown 转换工具

将 `.xlsx` 转为 UTF-8 的 Markdown 文件。

**表格格式：**

- 一律使用 **GFM 管道表格**（`| 列1 | 列2 |` + 分隔行），**不使用** `<table>` / `<tr>` / `<td>`。
- **有合并单元格** 时：只在 **合并区域左上角那一格** 写单元格内容，同一块合并里的其它格导出为 **空**。
- 导出后自动 **剔除全空行**（整行无可见文字）与 **全空列**（合并产生的冗余物理列），表格更紧凑。
- 自动识别 **表紙**（封面）、**表紙メタデータ区**（一覧 sheet 上部）与 **一覧表**，分别结构化输出。

**表紙 sheet** 输出结构示例：

```markdown
### 表紙

**譲渡 — 画面設計書**

#### プロジェクト情報
| 区分 | 内容 |
| プロジェクト | 林業業務システム更改に係る設計・開発 |
| サブシステム | 債務保証_平時 |
| 設計フェーズ | 基本設計 |
| 画面ID | SCR-A0300 |

#### 文書属性
| 項目 | 内容 |
| 文書番号 | … |
| バージョン | 0.90 |
| 作成者 | OKI |
| 作成日 | 2026/3/5 |
```

**空行：**

- 文档里 **连续多个空行会压成至多一行空行**。

依赖仅为 **Python 3.10+** 与 **`openpyxl`**（pip），不调用 LibreOffice 或 Excel。

---

## 安装

在 `ExcelToMd` 目录下执行：

```bash
pip install -r requirements.txt
```

注册为命令行工具（任意目录可用 `exceltomd`）：

```bash
pip install -e .
```

---

## 基本用法

**单个文件：**

```bash
python excel_to_md.py <输入.xlsx> -o <输出目录>
```

**文件夹批量（每个 xlsx 对应一个子文件夹）：**

```bash
python excel_to_md.py <输入文件夹> -o <输出目录>
```

例如 `input/` 下有 `A.xlsx`、`B.xlsx`，输出结构为：

```text
out/
  A/
    A__シート1.md
    A_images/
  B/
    B__シート1.md
    B_images/
```

子文件夹内再嵌套 xlsx 时，可加 `--recursive` 一并转换。

**默认：一个工作表对应一个 `.md` 文件**，文件名形如：

`工作簿名__Sheet1.md`、`工作簿名__设计書.md`

工作表名称中的非法路径字符会被替换为 `_`。

### 嵌入图片（默认开启）

- 输出目录下会创建 **`{工作簿名}_images/`**（与 `.md` 同级）。
- **文件名**：`图{工作表序号}-{图序号} {说明}.扩展名`  
  - 说明优先取 Excel 里图片的 **说明（descr）**，否则取 **名称（name）**；若为默认的 `Picture 1` 这类则省略说明，仅用 `图1-1.png` 这种形式。  
  - 非法文件名字符会替换为 `_`，过长会截断；重名会自动加 `_2`、`_3`。
- **Markdown**：该 sheet 的 `.md` 末尾增加「埋め込み画像」小节，用 `![説明](相对路径.png)` 直接引用图片文件（预览器可直接显示）。

不需要导出图片时加 `--no-export-images`。

---

## 命令行参数

| 参数 | 说明 |
|------|------|
| `input` | 必填。输入的 `.xlsx` 文件路径，或包含多个 `.xlsx` 的文件夹。 |
| `-o` / `--output-dir` | 必填。输出目录；若不存在会自动创建。文件夹批量时，每个 xlsx 在其下生成同名子文件夹。 |
| `--recursive` | 可选。文件夹输入时，递归处理子目录中的 `.xlsx`。 |
| `--one-file` | 可选。不加则「一表一文件」；加上后把所有工作表合并为**一个** `{工作簿名}.md`。 |
| `--no-shapes` | 可选。默认会解析形状/文本框内文字；加上后关闭。 |
| `--no-export-images` | 可选。默认会导出嵌入图片；加上后关闭。 |

终端里也可查看：

```bash
python excel_to_md.py --help
```

---

## 默认行为（重要）

- **单元格网格** 导出为管道表，自动去掉合并产生的全空行/列。
- **形状/文本框内文字** 默认从 DrawingML 抽取，追加为列表；**顺序不一定与画面一致**，也**不表示箭头/连线拓扑**。不需要时加 `--no-shapes`。
- 若重要说明**只写在文本框里**、没有写进单元格，可用 `--no-shapes` 关闭形状解析，或把内容挪到单元格。

---

## 编码与日语

- 输出的 `.md` 文件为 **UTF-8**，平假名、片假名、汉字等日语字符均可正常保存；现代编辑器与 Git 默认按 UTF-8 处理即可。

---

## 示例

```bash
# 每个 sheet 一个 md，默认含形状文字与嵌入图
python excel_to_md.py 设计书.xlsx -o ./out

# 文件夹内全部 xlsx，每个文件一个子文件夹
python excel_to_md.py ./xlsx_dir -o ./out

# 含子目录
python excel_to_md.py ./xlsx_dir -o ./out --recursive

# 合并为一个 md
python excel_to_md.py 设计书.xlsx -o ./out --one-file

# 只要单元格表格，不要形状文字和图片
python excel_to_md.py 设计书.xlsx -o ./out --no-shapes --no-export-images
```

成功时会在标准输出打印生成的每个 `.md` 及图片的完整路径。

---

## 限制摘要

| 内容 | 说明 |
|------|------|
| 合并单元格 | 管道表仅左上角一格有字；全空行列自动剔除；表紙情報与一覧自动拆分。Markdown 小节标题为日语。 |
| 文本框示意图 | 默认抽取文字；`--no-shapes` 可关闭。 |
| 箭头、流程拓扑 | Markdown 无法还原；形状文字顺序也不保证。 |
| 文件格式 | 仅 `.xlsx`。 |
| 嵌入图 | 默认导出；图表对象等未必在 `xl/media`，可能导不出。 |
